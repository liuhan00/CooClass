## 第一章 项目概述
### 1.1 产品简介
欢迎使用专注养成类微信小程序《咕咕学时》！这是一款将「专注效率」与「宠物养成」深度结合的轻量化工具，专为需要提升专注力、培养学习/工作习惯的用户打造。

用户可创建专属小鸡宠物，通过设定专注目标（如学习、工作、阅读等场景）并完成专注时长积累，获取「谷物币」和宠物经验值。小鸡形象采用软萌治愈风格，线条圆润可爱，拥有丰富的表情与动作反馈——专注时会乖巧看书、偷懒时会东张西望、升级后会解锁专属造型，让枯燥的专注过程变得更有陪伴感和成就感。

无需复杂操作，打开小程序即可快速开启专注，适配碎片化时间管理与长期习惯培养，帮助用户摆脱拖延，在轻松养成的乐趣中提升效率。

### 1.2 产品目标
1. 提供「高效专注+趣味养成」的双重体验，让用户主动坚持专注习惯。
2. 简化操作流程，降低使用门槛，适配学生、职场人士等多类用户的场景需求。
3. 设计多元化的养成体系与激励机制，提升产品粘性与用户留存率。
4. 打造清晰直观的数据统计功能，帮助用户可视化追踪专注成果。

### 1.3 项目价值定位
|           用户痛点             |                解决方案                   |          商业价值转化点           |
| ------------------------------ | ---------------------------------------- | -------------------------------- |
|        专注力不足，易被干扰     | 专注计时+小鸡陪伴反馈，专注期间屏蔽无效提醒 | 激励视频解锁额外奖励，提升广告曝光 |
|  坚持习惯缺乏动力，难以长期坚持  |     等级成长+造型解锁，阶段性目标激励      |  限定皮肤/道具内购，满足收藏需求   |
| 专注成果无记录，无法直观看到进步 |      多维度数据统计（日/周/月累计专注）    |   生成专属专注报告，支持分享裂变   |
|      碎片化时间利用效率低       |     支持10分钟-2小时灵活设定专注时长       |    高频次使用场景，提升用户活跃度  |

### 1.4 用户体验核心
设计「专注-奖励-成长-分享」的上瘾循环，强化用户正向反馈：
1. 专注完成 → 获得谷物币+经验值 → 小鸡升级/解锁新造型 → 增强成就感；
2. 连续专注 → 触发「连续专注礼包」 → 额外奖励稀有零食/限定装饰 → 激励长期坚持；
3. 数据达标 → 解锁成就勋章 → 生成分享海报 → 社交媒体炫耀 → 裂变新用户；
4. 偶尔中断 → 小鸡「失落表情」提示 → 触发用户不甘心心理 → 重新开启专注。

## 第二章 产品画风与美术设计
### 2.1 画风定位
1. 软萌治愈风：整体画面以温暖柔和的色调为主，主色调采用米白色+浅橙色，搭配淡绿色点缀，营造舒适放松的视觉体验；
2. 角色设定：
    - 小鸡主角：线条圆润，表情丰富（专注、开心、失落、好奇等），初始为黄色绒毛小鸡，升级后解锁不同造型（如戴学士帽、背书包、穿披风等），辨识度强；
    - 辅助元素：零食、装饰道具等采用卡通化设计，造型可爱，色彩鲜明，符合治愈系风格。

### 2.2 美术资源需求
1. 角色动画：
    - 小鸡基础动画：站立、进食、专注（看书/写字）、升级、失落、互动等动作；
    - 状态切换动画：专注开始/结束、升级特效、奖励领取等过渡动画；
2. 场景元素：
    - 专注场景：简洁的书桌背景、计时面板，支持切换主题（如森林、书桌、星空等）；
    - 养成场景：小鸡栖息地（初始为简单鸡窝，升级后解锁花园、书房等场景）；
3. UI界面：
    - 核心界面：主界面（小鸡展示+专注入口）、专注计时界面、零食商店界面、数据统计界面、成就界面；
    - 设计风格：简洁清晰，按钮圆润，图标卡通化，符合微信小程序轻量化操作体验。

## 第三章 产品玩法设计
### 3.1 核心玩法框架
核心逻辑：用户通过设定专注目标与时长 → 完成专注任务 → 获得奖励（谷物币+经验值） → 用谷物币购买零食 → 喂养小鸡提升等级 → 解锁新造型/新功能 → 数据统计与成就解锁 → 激励持续专注。

### 3.2 专注机制设计
1. 专注设定：
    - 时长选择：支持自定义时长（10分钟-2小时），同时提供预设模板（10分钟碎片专注、25分钟番茄专注、1小时深度专注）；
    - 场景标签：可选择专注场景（学习、工作、阅读、运动等），不同场景对应不同的小鸡专注动作（如学习场景小鸡看书，运动场景小鸡跑步）；
2. 专注过程：
    - 防干扰设计：专注开始后，界面仅显示计时进度与小鸡状态，屏蔽无关按钮，支持开启「专注模式」（手机系统级静音/免打扰提示）；
    - 中途退出惩罚：若未完成设定时长提前退出，无谷物币，无经验值，小鸡显示「失落表情」；
    - 连续专注奖励：连续3天完成专注，额外获得「连续专注礼包」（含稀有零食+双倍谷物币）；
3. 奖励规则：
    - 基础奖励：专注时长与奖励挂钩，1分钟专注=1谷物币，10分钟专注=10谷物币+1经验值（不足10分钟不计经验）；
    - 额外奖励：观看激励视频后，可获得当前专注奖励的1.5倍（谷物币+经验值）；
    - 邀请奖励：邀请好友注册并完成首次专注，双方各获得100谷物币+稀有零食1份。

### 3.3 小鸡养成体系
1. 等级成长：
    - 等级设定：共15级，初始为1级「绒毛小鸡」，每级所需经验值随等级提升递增（1-5级每级需50经验，6-10级每级需100经验，11-15级每级需200经验）；
    - 等级特权：
        - 1-5级：解锁基础表情、初始栖息地；
        - 6-10级：解锁更多专注场景、栖息地升级、普通装饰道具；
        - 11-15级：解锁限定造型、稀有装饰、专属专注特效；
2. 造型解锁：
    - 等级解锁：达到指定等级自动解锁基础造型（如3级解锁「红领巾小鸡」、8级解锁「学士帽小鸡」）；
    - 道具解锁：使用特定稀有零食喂养后，解锁限定造型（如「圣诞小鸡」「太空小鸡」）；
3. 小鸡状态：
    - 状态类型：正常、开心（刚喂食后）、专注、失落（中途退出专注）；
    - 状态影响：开心状态下，下次专注获得的经验值+20%；失落状态下，需完成1次完整专注才能恢复正常。

### 3.4 零食商店系统
1. 零食分类：
    - 普通零食：价格较低（50-200谷物币），喂养后仅增加经验值（10-30经验/份），如「小米粒」「玉米粒」；
    - 稀有零食：价格较高（500-1000谷物币）或通过活动/激励视频获得，喂养后增加大量经验值（50-100经验/份），并有概率解锁限定造型，如「彩虹谷粒」「金蛋饼干」；
    - 特殊零食：仅通过成就解锁或内购获得，喂养后可触发「双倍经验buff」（持续24小时），如「专注力饼干」；
2. 购买规则：
    - 普通零食无限量购买；
    - 稀有零食每日限购3份；
    - 特殊零食每月限购1份；
3. 商店更新：
    - 普通零食固定上架；
    - 稀有零食每周一更新1种限定款；
    - 节日期间推出专属零食（如春节「红包年糕」、中秋「月饼饼干」）。

### 3.5 数据统计与成就
1. 数据统计：
    - 核心数据：当日专注时长、累计专注时长、连续专注天数、专注场景分布；
    - 数据展示：日/周/月趋势图表，支持查看历史数据（最多保留3个月）；
    - 报告生成：每月自动生成「专注成长报告」，包含专注时长排名、进步幅度、个性化建议，支持分享至社交媒体；
2. 成就系统：

| 成就名称 | 解锁条件 | 成就描述 | 奖励内容 |
| ---- | ---- | ---- | ---- |
| 专注萌新 | 完成首次10分钟以上专注 | 开启专注之旅，小鸡为你加油！ | 50谷物币+普通零食1份 |
| 坚持使者 | 连续7天完成专注 | 坚持的力量，让习惯生根发芽 | 200谷物币+稀有零食1份 |
| 深度专注者 | 单次专注时长达到2小时 | 沉浸式专注，效率拉满！ | 300谷物币+「深度专注」勋章 |
| 学习达人 | 累计学习场景专注100小时 | 知识的积累，源于每一次专注 | 500谷物币+「学士帽」造型 |
| 社交达人 | 邀请5位好友注册并完成专注 | 分享专注的快乐，一起进步 | 1000谷物币+限定装饰「友谊围巾」 |
| 全场景大师 | 解锁所有专注场景并各完成1次专注 | 多元化专注，适配每一种需求 | 800谷物币+特殊零食1份 |
| 等级巅峰 | 小鸡等级达到15级 | 养成之路圆满，见证成长蜕变 | 2000谷物币+「巅峰小鸡」限定造型 |

### 3.6 创新体验优化
1. 个性化定制：
    - 支持更换专注背景主题（默认3种，额外2种需观看广告解锁）；
    - 自定义专注提示音（如小鸡咕咕声、雨声、白噪音等）；
2. 互动增强：
    - 专注完成后，小鸡会根据专注时长做出不同互动（如长时间专注会「献花」，短时间专注会「点头鼓励」）；
    - 累计专注达到一定时长，触发小鸡「隐藏剧情」（如讲述小鸡的成长故事）；
3. 降低迁移成本：
    - 支持导入其他专注类APP的历史专注数据（需用户授权），自动换算为累计专注时长，减少用户重新积累的顾虑；
    - 提供「新手引导任务」，完成后可快速获得大量谷物币，帮助新用户快速上手养成。

## 第四章 技术需求
### 4.1 前端技术
1. 开发框架：采用微信原生小程序开发框架（WXML+WXSS+JavaScript/TypeScript），确保小程序轻量化与流畅性；
2. 适配优化：支持不同屏幕尺寸（手机、平板）适配，界面元素自适应调整；
3. 动画处理：采用CSS3动画+小程序API动画结合，确保小鸡动作、升级特效等流畅无卡顿，文件大小控制在合理范围；
4. 存储策略：本地存储用户基础数据（当前专注进度、未使用的奖励），核心数据（累计专注时长、小鸡等级）实时同步至后端；
5. 性能优化：页面加载时间控制在2秒内，资源预加载策略，减少用户等待时间。

### 4.2 后端技术
1. 技术架构：采用SpringBoot + Mybatis-Plus + Redis + MySQL方案；
2. 核心功能处理：
    - 用户数据管理：存储用户信息、专注记录、小鸡养成数据、成就解锁状态等；
    - 计时验证：确保专注时长真实性，防止作弊（如后台运行时仍计算时长）；
    - AI适配：根据用户专注习惯（如常用时长、场景偏好），推荐个性化专注模板与零食；
3. 数据安全：用户数据加密存储，定期备份，确保数据不丢失；
4. 接口设计：提供稳定的前后端交互接口，支持高并发访问（峰值QPS≥1000）。

## 第五章 竞品分析
### 5.1 竞品选择
本次竞品分析选取三款同类型核心产品：《专注森林》《番茄TODO》《萌宠专注》。

### 5.2 竞品特点分析
| 产品名称 | 产品类型 | 主要特点 | 玩法特色 | 用户评价 |
| ---- | ---- | ---- | ---- | ---- |
| 《专注森林》 | 专注+种树养成 | 植物养成、社交打卡、场景化专注 | 专注完成后种植虚拟树木，可解锁不同植物，支持好友互动 | 画风治愈，激励性强，但养成体系较单一，广告较多 |
| 《番茄TODO》 | 专注工具+社区 | 强大的计时功能、自定义模板、学习社区 | 侧重效率工具属性，支持番茄工作法、锁机功能 | 功能全面，适合重度专注需求，但趣味性较弱，界面偏简洁 |
| 《萌宠专注》 | 专注+宠物养成 | 卡通宠物养成、简单奖励机制 | 专注获得积分，兑换宠物食物与装饰，养成体系基础 | 操作简单，萌系风格讨喜，但玩法单一，用户留存率一般 |

### 5.3 竞品启示
1. 优势借鉴：
    - 从《专注森林》借鉴「场景化养成+社交分享」模式，强化用户成就感与裂变传播；
    - 参考《番茄TODO》的「灵活计时+模板化」设计，满足不同用户的专注场景需求；
    - 延续《萌宠专注》的「轻量化养成」核心，避免过于复杂的玩法增加用户负担；
2. 差异化突破：
    - 强化「小鸡陪伴感」：相比竞品的植物/简单宠物，增加更多互动动作与剧情，提升情感连接；
    - 简化养成逻辑：避免竞品中复杂的资源体系，聚焦「专注-货币-喂养-升级」的核心循环，降低用户认知成本；
    - 解决用户痛点：针对竞品「广告过多」「养成动力不足」问题，控制广告频率（仅在解锁额外奖励时出现），设计阶段性目标激励（等级、造型、成就）；
    - 数据可视化：优化数据统计功能，提供更直观的趋势分析与个性化报告，帮助用户看到自身进步。

## 第六章 盈利性分析
### 6.1 主要盈利模式
1. 广告变现：
    - 激励视频广告：用户可通过观看广告解锁1.5倍奖励、稀有零食、限定背景等，广告时长控制在30秒内；
    - 插屏广告：在专注完成后、进入商店时等场景插入插屏广告，频率控制在「每3次操作出现1次」，避免过度干扰；
    - 横幅广告：在数据统计界面、成就界面底部展示横幅广告，样式简洁，不影响操作；
2. 内购变现：
    - 限定道具：推出节日限定小鸡造型、稀有装饰（如「天使翅膀」「魔法棒」），采用单次付费模式（价格5-20元）；
    - 增值服务：开通「会员特权」（月付10元/年付88元），享受无广告、双倍谷物币、专属零食无限购、自定义小鸡名称等权益；
    - 礼包套餐：推出「新手礼包」「成长礼包」，包含大量谷物币与稀有零食，满足用户快速养成需求（价格3-15元）。

### 6.2 盈利性分析
1. 低研发成本：微信小程序研发成本低，相比传统APP开发周期短（2-3个月即可上线），人力与时间成本可控；
2. 高用户潜力：专注类工具用户群体广泛（学生、职场人士、备考人群等），小程序无需下载，获客成本低，易形成规模效应；
3. 高转化效率：养成类玩法提升用户活跃度与留存率，高频次的专注场景为广告变现提供充足曝光机会，激励视频广告转化率预计高于普通工具类产品；
4. 可持续盈利：会员制与限定道具满足不同用户需求，随着用户规模扩大，广告收益与内购收益可同步增长。

### 6.3 盈利性风险
1. 用户体验与广告平衡：过多广告可能导致用户流失，需严格控制广告频率与展示场景，确保广告与用户需求（如解锁奖励）强关联；
2. 市场竞争风险：专注类产品较多，需快速打造差异化亮点（如小鸡互动、剧情化养成），避免陷入同质化竞争；
3. 内购转化风险：用户可能更倾向于免费养成，内购意愿不足，需优化限定道具与会员权益的吸引力，确保性价比；
4. 应对策略：
    - 建立用户反馈机制，根据用户意见调整广告频率与内购定价；
    - 持续更新内容（新造型、新场景、新成就），保持产品新鲜感；
    - 推出「免费试用会员」「限时折扣礼包」，降低用户内购决策门槛。
